# WaveFrame Beta Bug Reporting

Use GitHub issues or the manual tracker in `BETA_FEEDBACK_TRACKER.md` for private beta reports.

## Preferred Issue Templates

- `Beta bug report`: crashes, audio pass-through issues, DAW issues, output/OBS issues, Frame/HUD issues, and module/template bugs.
- `Beta feedback`: UX confusion, visual quality feedback, documentation notes, and polish feedback.
- `Feature request`: ideas that should be considered for a future beta or roadmap.

## Required Details

Ask testers for:
- Artifact/run number.
- Windows version.
- Standalone or VST3.
- DAW and DAW version, if relevant.
- Audio interface, if relevant.
- Selected module and template.
- Output window open: yes/no.
- OBS involved: yes/no.
- Steps to reproduce.
- Expected behavior.
- Actual behavior.
- Repro rate.
- Screenshot, video, or log when useful.
- Whether audio pass-through changed.
- Whether the app, plugin, or DAW crashed/froze.

## Handling Reports

- Treat audio pass-through changes as P0 until disproven.
- Mark incomplete reports `Needs info`.
- Use `FEEDBACK_TRIAGE_RULES.md` for severity and decisions.
- Use `FIX_SPRINT_WORKFLOW.md` for focused bug-fix PRs.
- Move future ideas to roadmap/TODO instead of the current fix sprint.
